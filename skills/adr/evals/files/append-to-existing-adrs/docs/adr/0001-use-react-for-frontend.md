# 0001. Use React for Frontend

- Status: Accepted
- Date: 2025-01-15

## Context and drivers

The team needs to choose a frontend framework for the new customer portal. We need component reuse, a large ecosystem, and team familiarity.

## Considered options

- React - widely used, large ecosystem, team has experience
- Vue.js - simpler API, good documentation
- Angular - full framework, steeper learning curve

## Decision

We will use React. The team has the most experience with it and the ecosystem of libraries for our needs (charting, forms, state management) is mature.

## Consequences

- Positive: Fast onboarding for new developers familiar with React.
- Positive: Rich ecosystem of third-party libraries.
- Negative: More boilerplate than Vue for simple components.
